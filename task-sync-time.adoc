= NTPサーバーを使用してCloud Volumes ONTAPシステム時間を同期する
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
Cloud Volumes ONTAPクラスタのネットワークタイムプロトコル（NTP）サーバーを設定すると、ネットワーク内の他のシステムや外部サーバーとクラスタ時間を正確に同期できます。クラスタ時間を外部NTPサーバーと同期することで、インフラストラクチャ全体の一貫性を維持できます。NetAppは、新しいCloud Volumes ONTAPデプロイメントに対してデフォルトでNTPサーバーを設定します。ただし、既存のCloud Volumes ONTAPクラスタに対して設定されていない場合は、ネットワーク内外で正確な時刻同期を実現するためにNTPサーバーを設定する必要があります。

NTP サーバーは次のように指定できます。

* https://docs.netapp.com/us-en/bluexp-automation/cm/api_ref_resources.html["NetApp ConsoleAPI"^]。
* ONTAP CLIコマンド https://docs.netapp.com/us-en/ontap-cli//cluster-time-service-ntp-server-create.html["cluster time-service ntp server create"^]。



NOTE: NTP サーバーを構成しないと、サービスが中断され、時刻の同期が不正確になる可能性があります。
